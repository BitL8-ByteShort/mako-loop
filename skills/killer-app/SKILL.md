# Killer App

Use this mode pack with `mako-loop`. Read `../../references/killer-app.md` before building or judging a serious app workflow.

## Core Rule

Do not build a platform shell. Finish one operational workflow so a real user can understand state, take action, and see the result.

## Required Proof

- One end-to-end workflow passes.
- Empty, loading, error, and success states are coherent.
- Primary nav reflects real product surfaces only.
- No diagnostics masquerade as product UI.
- Browser-visible proof exists.
